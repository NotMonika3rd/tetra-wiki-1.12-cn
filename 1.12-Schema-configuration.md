模式定义了为特定插槽制作特定模块所需的材料和工具。可用的模式在位于[modules resource folder](/mickelus/tetra/blob/master/src/main/resources/data/tetra/modules)的json文件中定义 .

每个 schema 配置文件可以通过在 config 文件夹中放置同名文件来替换。例如， [heavy_blade.json](/mickelus/tetra/blob/master/src/main/resources/data/tetra/schemas/sword/heavy_blade.json) 可以通过在 minecraft 文件夹中创建文件 `config/tetra/schemas/sword/heavy_blade.json` 来替换。

schema 配置文件包含 json 对象数组，每个对象描述一个单独的 schema。有关 schema 字段的更多详细信息，请参见 [SchemaDefinition.java](/mickelus/tetra/blob/master/src/main/java/se/mickelus/tetra/module/schema/SchemaDefinition.java) 和 [OutcomeDefinition.java](/mickelus/tetra/blob/master/src/main/java/se/mickelus/tetra/module/schema/OutcomeDefinition.java)，或查看 [schema configurations folder](/mickelus/tetra/blob/master/src/main/resources/data/tetra/schemas) 中的 json 文件以获取示例。

添加新 schema 时，必须创建/更新以下资产：
* 与 schema 键匹配的本地化条目（schema 名称、schema 描述以及每个材料插槽的标签）
