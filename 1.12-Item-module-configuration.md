模块物品有多个插槽，玩家可以将不同类型和变体的模块放入其中。可用的模块在位于 [modules resource folder](/mickelus/tetra/blob/1.12/src/main/resources/data/tetra/modules) 的 json 文件中定义。

每个模块配置文件可以通过在配置文件夹中放置同名文件来替换。例如， [heavy_blade.json](/mickelus/tetra/blob/1.12/src/main/resources/data/tetra/modules/sword/heavy_blade.json) 可以通过在 minecraft 文件夹中创建文件 `config/tetra/modules/sword/heavy_blade.json` 来替换。

模块插槽有两种不同类型：主要和次要。放在主要插槽中的模块使用较大的图标（16x16 对比 9x9），但在不同插槽中放置的模块提供的属性和效果没有区别。

模块配置文件包含 json 对象的数组，这些 json 对象中的字段和数值影响模块将为物品添加哪些属性和行为。有关可用字段及其作用的更多详细信息，请参见 [ModuleData.java](/mickelus/tetra/blob/1.12/src/main/java/se/mickelus/tetra/module/data/ModuleData.java)，或查看 [modules resource folder](/mickelus/tetra/blob/1.12/src/main/resources/data/tetra/modules) 中的 json 文件以获取示例。

主要模块插槽也可以容纳改进，它们的配置方式与模块相同，但有一些额外的字段（见 [ImprovementData.java](/mickelus/tetra/blob/1.12/src/main/java/se/mickelus/tetra/module/data/ImprovementData.java)），并位于名为 `improvements` 的文件夹中。改进配置文件的名称与其适用的模块相同，除了 [blade_enchants.json](/mickelus/tetra/blob/1.12/src/main/resources/data/tetra/modules/sword/improvements/blade_enchants.json) 之外。定义在 blade_enchants.json 中的改进适用于所有刀刃。

有关模块能力的信息，请参见 [Capability.java](/mickelus/tetra/blob/1.12/src/main/java/se/mickelus/tetra/capabilities/Capability.java)，有关物品效果的信息，请参见 [ItemEffect.java](/mickelus/tetra/blob/1.12/src/main/java/se/mickelus/tetra/module/ItemEffect.java)。

模块配置文件仅定义哪些模块存在，模式定义随后用于定义玩家可以在何种插槽和条件下制作模块。

添加新模块（或现有模块的新变体，例如小麦短刀）时，必须创建/更新以下资产：
* 一个放置在 [textures folder](/mickelus/tetra/blob/1.12/src/main/resources/data/tetra/modules) 中的纹理，其名称与新模块定义中的键字段匹配（在资源包中）

* 与模块键匹配的本地化条目（模块名称、物品名称 + 前缀（如需要））
